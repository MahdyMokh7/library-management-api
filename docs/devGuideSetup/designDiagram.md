## 📋 Summary of Design Decisions

| Aspect | Decision | Rationale |
|--------|----------|-----------|
| **Architecture** | Layered (Controller → Service → Repository) | Separation of concerns, SOLID principles |
| **Business Logic** | Service layer with interfaces | Dependency Inversion, testability |
| **Validation** | Bean Validation + Custom Validators | Input validation + Business rule enforcement |
| **Dynamic Queries** | JPA Specifications | Type-safe, flexible filtering (Open/Closed) |
| **Mapping** | MapStruct | Compile-time mapping, no reflection overhead |
| **Transaction** | @Transactional in Service layer | Ensures ACID properties |
| **Soft Delete** | isDeleted flag + @Where | Data retention, GDPR compliance |
| **Concurrency** | @Version optimistic locking | Prevent lost updates |
| **Exception** | Global @ControllerAdvice | Consistent error responses |
| **DTOs** | Separate Request/Response | API versioning, security |

This design fully meets the official document requirements while incorporating professional-grade practices like Docker, CI/CD, and advanced patterns.
